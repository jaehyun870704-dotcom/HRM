# Quiz Stat

`hrm-quiz/src/data/questions.ts`와 관련 소스 파일을 분석해 퀴즈 게임의 통계를 출력합니다.

## 수행 순서

### 1단계: 파일 읽기

아래 파일을 읽는다.
- `hrm-quiz/src/data/questions.ts`
- `hrm-quiz/src/types/index.ts`

### 2단계: 문제 데이터 통계

#### 전체 현황
```
총 문제 수: N
```

#### 카테고리별
```
HRM실무:   N문제 (XX%)
근로관계법: N문제 (XX%)
```

#### 출처별
```
기출: N문제
  └ 제40회: N문제
  └ 제41회: N문제
이론: N문제
```

#### 난이도별
```
쉬움:   N문제 (XX%)
보통:   N문제 (XX%)
어려움: N문제 (XX%)
```

#### 카테고리 × 난이도 교차표
```
              쉬움    보통    어려움    합계
HRM실무         N       N        N       N
근로관계법       N       N        N       N
합계            N       N        N       N
```

#### 정답 분포
```
① N문제 (XX%)  ② N문제 (XX%)  ③ N문제 (XX%)  ④ N문제 (XX%)
```

#### 텍스트 길이 분석
- 문제 텍스트 평균 길이: N자
- 가장 긴 문제 [id=N]: N자
- 가장 짧은 문제 [id=N]: N자
- 보기 평균 길이: N자

### 3단계: 데이터 품질 지표

```
필드 완성도:
  explanation 작성률:      XX%
  wrongExplanation 작성률: XX%
  originalNumber 기입률:   XX% (기출 문제 기준)

잠재적 이슈:
  빈 필드가 있는 문제: N건
  options 개수 이상:   N건
```

### 4단계: 개선 제안

통계 결과를 바탕으로 아래 기준에 해당하면 제안을 출력하세요.

- 카테고리 비율이 7:3 이상으로 편중 → "⚠️ 카테고리 불균형: {카테고리} 문제 추가 권장"
- 어느 한 난이도(쉬움/보통/어려움)가 전체의 70% 이상 → "⚠️ 난이도 불균형: {난이도} 비율 과다, 다른 난이도 문제 추가 권장"
- 쉬움 또는 보통이 0개 → "⚠️ 난이도 공백: {난이도} 문제 없음, 추가 권장"
- 정답 분포에서 특정 번호가 40% 이상 → "⚠️ 정답 편중: ①~④ 중 {번호} 정답 비율 과다"
- 총 문제 수가 20 미만 → "⚠️ 문제 수 부족: /quiz-add 로 문제 추가 권장"
- 이슈 없으면 → "✅ 데이터 품질 양호"
