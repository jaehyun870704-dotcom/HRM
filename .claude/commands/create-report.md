# Create Report

전체 응시자의 점수를 **상위 비율 기준 상대평가**로 등급을 산출하고, 성적 보고서를 생성합니다.

## 등급 기준 (상대평가)

| 등급 | 기준 |
|------|------|
| A | 상위 20% 이내 |
| B | 상위 21% ~ 40% |
| C | 상위 41% ~ 70% |
| D | 하위 30% (상위 71% 이하) |

> 동점자는 같은 등급을 부여하고, 다음 등급 구간은 동점자 수만큼 밀립니다.

## 수행 순서

### 1단계: 데이터 읽기

아래 파일을 읽는다.
- `hrm-quiz/src/hooks/useQuiz.ts` — localStorage 키(`hrm-quiz-rankings`) 및 `RankRecord` 저장 방식 확인
- `hrm-quiz/src/types/index.ts` — `RankRecord` 타입 확인

### 2단계: 상대평가 등급 계산 방법 확인

읽은 데이터를 바탕으로 아래 로직으로 등급을 계산합니다.

```
1. 전체 응시자를 점수 내림차순 정렬 (동점 시 소요시간 오름차순)
2. 각 응시자의 백분위 계산:
     percentile = (순위 - 1) / 전체 인원 * 100
3. 등급 부여:
     percentile < 20  → A
     percentile < 40  → B
     percentile < 70  → C
     그 외             → D
4. 동점자 처리: 동점자 중 가장 높은 순위의 백분위를 그룹 전체에 적용
```

### 4단계: 예외 처리

아래 상황에 따라 다음 메시지를 출력하세요.

- 응시자 0명: `⚠️ 응시자 데이터가 없습니다. 학생들이 퀴즈를 완료한 후 다시 실행하세요.`
- 응시자 1명: `⚠️ 응시자가 1명입니다. 상대평가는 2명 이상부터 의미가 있습니다. 해당 응시자는 자동으로 A등급 처리합니다.`
- 전원 동점: `⚠️ 전원 동점입니다. 모든 응시자에게 A등급을 부여합니다.`

### 5단계: 코드 반영 여부 확인

보고서 출력 후 아래를 질문하세요.

> "상대평가 등급 기준을 `TeacherDashboard.tsx` 및 `useQuiz.ts`의 `calcGrade` 로직에도 반영할까요?
> 반영하면 퀴즈 결과 화면과 선생님 대시보드의 등급 표시가 상대평가 기준으로 변경됩니다."

- 사용자가 **예** 라고 답하면:
  1. `hrm-quiz/src/hooks/useQuiz.ts`의 `calcGrade` 함수를 상대평가 로직으로 교체
  2. `hrm-quiz/src/components/TeacherDashboard.tsx`의 등급 계산 로직 동기화
  3. `npx tsc --noEmit` 및 `npm run lint` 실행 후 결과 보고

- 사용자가 **아니오** 라고 답하면: 보고서 출력만으로 종료
