# Quiz Leaderboard

퀴즈 게임의 순위 시스템(`RankRecord`)을 관리합니다.

## 수행 순서

### 1단계: 소스 파악

아래 파일을 읽는다.
- `hrm-quiz/src/types/index.ts` — `RankRecord` 타입 확인
- `hrm-quiz/src/` 하위 전체 — localStorage 키, 랭킹 저장·조회 로직 탐색

`RankRecord` 구조 (참고):
```ts
interface RankRecord {
  name: string;
  score: number;
  total: number;
  grade: string;
  elapsedSeconds: number;
  playedAt: string;
}
```

### 2단계: 현재 구현 상태 진단

탐색 결과를 바탕으로 아래 항목의 구현 여부를 표로 출력하세요.

```
기능                         상태
─────────────────────────────────────────
랭킹 저장 (localStorage)     ✅ 구현 / ❌ 미구현 / ⚠️ 부분구현
랭킹 불러오기                ✅ / ❌ / ⚠️
랭킹 정렬 (점수·시간 기준)   ✅ / ❌ / ⚠️
랭킹 상위 N개 제한           ✅ / ❌ / ⚠️
랭킹 초기화                  ✅ / ❌ / ⚠️
랭킹 UI 표시                 ✅ / ❌ / ⚠️
```

### 3단계: 작업 선택

구현 상태에 따라 아래 중 적합한 작업을 수행하세요.

#### A. 미구현 항목이 있는 경우 — 구현 제안

누락된 기능 목록과 함께 구현 방향을 제시하세요.

```
미구현 기능: <목록>

권장 구현 방향:
1. localStorage 키: 'hrm-quiz-rankings'
2. 저장 형식: JSON.stringify(RankRecord[])
3. 정렬 기준: score 내림차순 → elapsedSeconds 오름차순 (동점 시 빠른 시간 우선)
4. 최대 보관: 상위 10개
5. grade 기준: 예) 90점↑ S등급 / 70점↑ A등급 / 50점↑ B등급 / 그 이하 C등급

구현이 필요한 파일:
  - <파일 경로>: <추가·수정 내용>
```

사용자 확인 후 구현할지 물어보세요: "위 내용으로 구현을 진행할까요?"

#### B. 모두 구현된 경우 — 현황 리포트

```

관련 파일:
  - <파일>:<라인> — <역할>
```

### 4단계: 개선 제안

현재 구현(또는 제안 구현)에 대해 아래 관점에서 개선점을 제시하세요.

- **동점 처리**: 점수가 같을 때 elapsedSeconds로 구분하는지 여부
- **중복 이름**: 동일 `name`으로 여러 기록이 저장될 때 처리 방식
- **데이터 유효성**: `score > total` 또는 `elapsedSeconds < 0` 방어 코드 여부
- **UX**: 자신의 순위 하이라이트, 신기록 알림 등

개선 필요 없으면 `✅ 개선 사항 없음` 을 출력하세요.
