# Quiz Add

`hrm-quiz/src/data/questions.ts`에 새 문제를 추가합니다.

- **$1** — 카테고리: `HRM실무` | `근로관계법`
- **$2** — 난이도: `쉬움` | `보통` | `어려움`

## 수행 순서

### 1단계: 인수 검증

$1이 `HRM실무` 또는 `근로관계법`이 아니거나, $2가 `쉬움`, `보통`, `어려움` 중 하나가 아니면 즉시 중단하고 아래를 출력하세요.

```
❌ 사용법: /quiz-add <카테고리> <난이도>
   카테고리: HRM실무 | 근로관계법
   난이도:   쉬움 | 보통 | 어려움
```

### 2단계: 현재 파일 파악

`hrm-quiz/src/data/questions.ts`를 읽고 다음을 확인하세요.

- 현재 마지막 `id` 값 → 새 문제의 `id = 마지막 id + 1`
- 파일 끝 `];` 위치 → 새 문제를 그 바로 앞에 삽입

### 3단계: 문제 생성

카테고리 **$1**, 난이도 **$2**에 맞는 새 문제를 직접 작성하세요.
아래 **검증 가이드라인**을 반드시 준수합니다.

#### 검증 가이드라인

| 필드 | 규칙 |
|------|------|
| `id` | 기존 최대 id + 1 (중복 불가) |
| `source` | `'기출'` 또는 `'이론'` |
| `exam` | `source === '기출'`이면 `'제40회' \| '제41회'`, `source === '이론'`이면 반드시 `null` |
| `originalNumber` | `source === '기출'`이면 양의 정수, `source === '이론'`이면 반드시 `null` |
| `category` | 인수 $1 그대로 사용 |
| `difficulty` | 인수 $2 그대로 사용 |
| `options` | 정확히 4개의 문자열, 빈 문자열 불가 |
| `answer` | `0 \| 1 \| 2 \| 3` (0-index), 빈 문자열 불가 |
| `explanation` | 정답 이유를 구체적으로 서술, 빈 문자열 불가 |
| `wrongExplanation` | 오답 포인트를 구체적으로 서술, 빈 문자열 불가 |

#### 기존 파일과 동일한 코드 형식

```ts
  {
    id: N,
    source: '이론',
    exam: null,
    originalNumber: null,
    category: '$1',
    difficulty: '$2',
    question:
      '문제 내용',
    options: [
      '보기1',
      '보기2',
      '보기3',
      '보기4',
    ],
    answer: 0,
    explanation:
      '정답 해설',
    wrongExplanation:
      '오답 포인트',
  },
```

### 4단계: 파일 수정

`hrm-quiz/src/data/questions.ts`의 닫는 `];` 바로 앞에 새 문제 객체를 삽입하세요.
기존 코드는 절대 변경하지 않습니다.

### 5단계: 삽입 후 검증

삽입 후 아래 항목을 재확인하고 결과를 출력하세요.

```
✅ 문제 추가 완료
   id: N
   카테고리: $1
   난이도: $2
   출처: 기출/이론
   정답: ①~④
   현재 총 문제 수: N
```

오류가 발견되면 파일을 원래대로 되돌리고 `❌ 추가 실패: <오류 내용>` 을 출력하세요.
