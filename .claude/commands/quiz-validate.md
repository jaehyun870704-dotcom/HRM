# Quiz Validate

`$ARGUMENTS`가 입력된 경우 해당 카테고리(`HRM실무` 또는 `근로관계법`)의 문제만 검증합니다.
입력이 없으면 전체 문제를 검증합니다.

## 실행 전 준비

다음 파일을 읽으세요.
- `hrm-quiz/src/data/questions.ts`
- `hrm-quiz/src/components/StartScreen.tsx`
- `hrm-quiz/src/components/NextButton.tsx`

### PDF 텍스트 추출 도구
`pdftotext`가 설치되어 있어 기출문제 자동 대조가 가능합니다.
- 제40회: `pdftotext "D:/vibecoding/practice/제40회_HRM전문가 문제지.pdf" -`
- 제41회: `pdftotext "D:/vibecoding/practice/제41회_HRM전문가 문제지.pdf" -`

추출 텍스트는 인코딩 손실이 있으므로 핵심 키워드·번호·보기 순서 위주로 대조하세요.

`$ARGUMENTS`가 비어있지 않으면 `questions` 배열에서 `category === '$ARGUMENTS'`인 문제만 대상으로 삼으세요.
`$ARGUMENTS`가 비어있으면 전체 문제를 대상으로 삼으세요.

## 검증 항목

### 항목 1. 기출문제 원본 대조 (제40회 · 제41회)

대상 문제 중 `source === '기출'`인 문제를 회차별로 분류하여 아래 표 형식으로 출력하세요.

```
| id | 회차 | 원번호 | 문제 앞 40자 | 보기 수 |
```

#### PDF 자동 대조 절차
1. `pdftotext`로 해당 회차 PDF 텍스트를 추출합니다.
2. 원번호에 해당하는 문제 텍스트를 찾아 핵심 키워드·보기 순서를 비교합니다.
3. 판정 기준:
   - 문제·보기의 핵심 내용이 일치하면: `✅ [id=N] 제N회 N번 — 원본 대조 일치`
   - 불일치 또는 인코딩 손실로 불확실하면: `⚠️ [id=N] 제N회 N번 — 육안 재확인 권장`

### 항목 2. 이론문제 HRM 교재 기준 여부

대상 문제 중 `source === '이론'`인 문제를 아래 표 형식으로 출력하세요.

```
| id | category | 문제 전체 |
```

그런 다음 아래 기준으로 판단하세요.
- HRM 전문가 교재의 개념·용어와 일치해야 합니다.
- 자동 판단 불가 시: `⚠️ [id=N] 이론 문제 — HRM 전문가 교재와 육안 대조 필요`

해당 문제가 없으면: `ℹ️ 대상 범위에 이론 문제 없음`

---

### 항목 3. 첫 화면 카테고리 표기

`StartScreen.tsx`에서 확인하세요.

| 확인 내용 | 기준 | 결과 |
|-----------|------|------|
| "친밀관계법" 문자열 존재 여부 | 없어야 함 | ✅ / ❌ |
| "근로관계법" 문자열 존재 여부 | 있어야 함 | ✅ / ❌ |
| "총 N문제" 안내 텍스트 존재 여부 | 있어야 함 | ✅ / ⚠️ |

오류 형식: `❌ [StartScreen] "친밀관계법" 표현 발견 → "근로관계법"으로 수정 필요`

---

### 항목 4. 정답 유일성

대상 문제 각각에 대해 확인하세요.

- `options[0~3]` 중 **동일한 문자열**이 2개 이상이면: `❌ [id=N] 보기 중복: "..." 동일 항목 존재`
- `options[answer]`(정답 보기)와 나머지 보기 사이에 **의미가 동일하거나 거의 유사**한 표현이 있으면: `⚠️ [id=N] 정답과 유사한 보기 존재 가능 — 검토 필요`
- 전체 이상 없으면: `✅ 정답 유일성 확인됨`

---

### 항목 5. 난이도 3단계 구분

대상 문제 각각의 `difficulty` 값을 확인하세요.

- 허용 값: `'쉬움'` | `'보통'` | `'어려움'`
- 허용 외 값(`'Easy'`, `'Hard'`, `'Normal'` 등) 발견 시: `❌ [id=N] difficulty 값 오류: "실제값" → '쉬움'|'보통'|'어려움' 중 하나로 수정 필요`
- 3단계 중 하나도 사용되지 않은 레벨이 있으면: `⚠️ 난이도 "쉬움/보통/어려움" 중 비어있는 단계 존재`
- 이상 없으면: `✅ 난이도 3단계 정상`

난이도별 문제 수도 출력하세요.
```
난이도별: 쉬움 N / 보통 N / 어려움 N
```

---

### 항목 6. 다음 문제 버튼

`NextButton.tsx`에서 확인하세요.

| 확인 내용 | 기준 | 결과 |
|-----------|------|------|
| 파일 존재 여부 | 있어야 함 | ✅ / ❌ |
| 문제 풀기 전 버튼 숨김 | `isAnswered === false`일 때 렌더링 안 함 | ✅ / ❌ |
| 마지막 문제 이후 결과 화면 이동 | `isLast` 분기 처리 | ✅ / ❌ |

오류 형식: `❌ [NextButton] isLast 분기 없음 — 마지막 문제 후 결과 이동 불가`
