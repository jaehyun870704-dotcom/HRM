# Quiz Daily

퀴즈 데이터의 일일 점검 및 문제 보충 작업을 수행합니다.
각 단계가 실패하면 즉시 중단하고 오류를 보고합니다.

---

## STEP 1 — 파일 구조 파악

다음 파일을 순서대로 읽는다.

1. `hrm-quiz/src/types/index.ts` — `Question`, `RankRecord`, 타입 전체 파악
2. `hrm-quiz/src/data/questions.ts` — 문제 배열 전체 파악

읽기 실패 시:
```
❌ [STEP 1 실패] 파일을 읽을 수 없습니다: <경로>
→ 작업을 중단합니다.
```

성공 시 출력:
```
✅ STEP 1 완료 — 파일 구조 파악
   Question 필드 수: N개
   questions 배열 항목 수: N개
```

---

## STEP 2 — 현재 문제 개수와 분포 확인

STEP 1에서 읽은 데이터를 기반으로 아래 통계를 계산한다.

```
총 문제 수: N

카테고리별:
  HRM실무:    N문제 (XX%)
  근로관계법:  N문제 (XX%)

난이도별:
  쉬움:   N문제 (XX%)
  보통:   N문제 (XX%)
  어려움: N문제 (XX%)

출처별:
  기출: N문제 (제40회 N, 제41회 N)
  이론: N문제

정답 분포:
  ① N  ② N  ③ N  ④ N
```

통계 계산 중 예외 발생 시:
```
❌ [STEP 2 실패] 통계 계산 오류: <오류 내용>
→ 작업을 중단합니다.
```

성공 시:
```
✅ STEP 2 완료 — 현재 분포 확인
```

---

## STEP 3 — 카테고리별 부족한 부분 파악

아래 기준으로 부족 항목을 판정한다.

| 판정 기준 | 임계값 |
|-----------|--------|
| 카테고리 불균형 | 두 카테고리 비율이 6:4 이상 편중 |
| 난이도 불균형 | 어느 한 난이도(쉬움/보통/어려움)가 70% 초과 또는 어느 한 단계가 0개 |
| 정답 편중 | 특정 번호가 전체의 40% 이상 |
| 문제 수 부족 | 총 문제 수 20개 미만 (현재 21개, 기준 유지) |
| 카테고리별 최소 | 어느 카테고리든 8문제 미만 |

출력 형식:
```
부족 항목 분석:
  ⚠️ <항목>: <현재 상태> → <권장 상태>
  ...
  이슈 없으면: ✅ 모든 분포 양호
```

이후 **오늘 추가할 문제 목표**를 제시한다:
```
오늘 추가 목표:
  HRM실무   쉬움:   N문제
  HRM실무   보통:   N문제
  HRM실무   어려움: N문제
  근로관계법 쉬움:   N문제
  근로관계법 보통:   N문제
  근로관계법 어려움: N문제
  합계: N문제
  (부족 항목 없으면 "추가 불필요 — 선택적으로 1문제 추가 가능" 출력)
```

성공 시:
```
✅ STEP 3 완료 — 부족 항목 파악
```

## STEP 4 — 새 문제 추가 전 중복 체크

추가 예정 문제(STEP 3 목표 기준)를 작성하기 전에 아래를 검사한다.

#### 4-1. id 중복 방지
- 현재 최대 id를 확인하고 신규 id = 최대 id + 1부터 순차 부여

#### 4-2. 문제 내용 중복 방지
- 추가할 문제의 핵심 키워드가 기존 `question` 필드에 이미 존재하는지 확인
- 80% 이상 유사한 문제가 있으면 해당 문제를 건너뛰고 경고

#### 4-3. 기출 문제 번호 중복 방지
- `source === '기출'`인 경우 동일 `exam` + `originalNumber` 조합이 이미 존재하는지 확인

출력:
```
중복 체크 결과:
  사용 가능한 시작 id: N
  내용 유사 문제: N건 (있으면 목록 출력)
  기출 번호 충돌: N건 (있으면 목록 출력)
```

중복 체크 실패(로직 오류) 시:
```
❌ [STEP 4 실패] 중복 체크 중 오류: <오류 내용>
→ 작업을 중단합니다.
```

성공 시:
```
✅ STEP 4 완료 — 중복 체크 이상 없음
```

## STEP 5 — 문제 추가 및 형식 검증

STEP 3 목표에 따라 문제를 작성하고 `hrm-quiz/src/data/questions.ts`에 삽입한다.

#### 5-1. 문제 작성 규칙

각 문제는 아래 검증 가이드라인을 반드시 준수한다:

| 필드 | 규칙 |
|------|------|
| `id` | STEP 4에서 확인한 시작 id부터 순차 부여 |
| `source` | `'기출'` 또는 `'이론'` |
| `exam` | `'기출'`이면 `'제40회' \| '제41회'`, `'이론'`이면 `null` |
| `originalNumber` | `'기출'`이면 양의 정수, `'이론'`이면 `null` |
| `options` | 정확히 4개, 빈 문자열 불가 |
| `answer` | `0 \| 1 \| 2 \| 3` (0-index) |
| `explanation` | 정답 이유 구체적 서술, 빈 문자열 불가 |
| `wrongExplanation` | 오답 포인트 구체적 서술, 빈 문자열 불가 |

#### 5-2. 삽입 위치

`questions.ts`의 닫는 `];` 바로 앞에 추가한다. 기존 코드는 절대 수정하지 않는다.

#### 5-3. 삽입 후 즉시 검증

삽입 직후 아래를 재확인한다:
- 파일이 유효한 TypeScript 배열 형태인지 (`[` 로 시작, `];` 로 종료)
- 추가된 문제의 모든 필드가 타입에 맞는지
- id 중복이 없는지

검증 실패 시 삽입한 내용을 즉시 롤백하고:
```
❌ [STEP 5 실패] 형식 검증 오류 — 삽입 내용을 롤백했습니다.
   오류: <상세 내용>
→ 작업을 중단합니다.
```

성공 시:
```
✅ STEP 5 완료 — 문제 추가 및 형식 검증
   추가된 문제: N개 (id N~N)
```

## STEP 6 — 전체 데이터 백업

`hrm-quiz/src/data/questions.ts`의 현재 내용을 아래 경로에 백업한다.

```
hrm-quiz/src/data/questions.backup.YYYYMMDD.ts
```

- `YYYYMMDD`는 오늘 날짜 (예: 20260329)
- 동일 날짜 백업이 이미 존재하면 덮어쓰기 전에 알린다:
  ```
  ⚠️ 오늘자 백업이 이미 존재합니다: questions.backup.YYYYMMDD.ts
     덮어씁니다.
  ```

백업 실패 시:
```
❌ [STEP 6 실패] 백업 파일 생성 실패: <오류 내용>
→ 작업을 중단합니다.
```

성공 시:
```
✅ STEP 6 완료 — 백업 생성
   백업 경로: hrm-quiz/src/data/questions.backup.YYYYMMDD.ts
   백업 문제 수: N개
```

## STEP 7 — 실행 결과 상세 보고

모든 단계 완료 후 최종 리포트를 출력한다.

```
══════════════════════════════════════
  📋 Quiz Daily 실행 결과 보고
  실행일: YYYY-MM-DD
══════════════════════════════════════

[ 단계별 결과 ]
  STEP 1  파일 구조 파악       ✅
  STEP 2  분포 확인            ✅
  STEP 3  부족 항목 파악       ✅
  STEP 4  중복 체크            ✅
  STEP 5  문제 추가·검증       ✅
  STEP 6  백업                 ✅

[ 문제 변동 ]
  작업 전: N문제
  추가:   +N문제
  작업 후: N문제

[ 추가된 문제 목록 ]
  [id=N] 카테고리/난이도 — 문제 앞 30자...
  ...

[ 작업 후 분포 ]
  카테고리: HRM실무 N / 근로관계법 N
  난이도:   쉬움 N / 보통 N / 어려움 N
  정답:     ① N  ② N  ③ N  ④ N

[ 백업 ]
  경로: hrm-quiz/src/data/questions.backup.YYYYMMDD.ts

══════════════════════════════════════
✅ Quiz Daily 완료
══════════════════════════════════════
```

중간 단계에서 실패한 경우:
```
══════════════════════════════════════
  ❌ Quiz Daily 중단 보고
  실행일: YYYY-MM-DD
══════════════════════════════════════
  실패 단계: STEP N — <단계명>
  오류 내용: <상세 메시지>
  롤백 여부: 완료 / 불필요
  파일 상태: 변경 없음 / 롤백 완료
══════════════════════════════════════
```
